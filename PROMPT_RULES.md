# PROMPT_RULES.md

# app_hr Prompt 设计规范

Version: 1.0

## 1. 目标

Prompt 的职责是帮助 LLM 理解用户需求、理解数据结构、生成安全可执行的
Python 代码，并解释执行结果。

原则：

-   Prompt 尽可能短。
-   不发送完整数据。
-   不要求 LLM 自行计算。
-   输出格式固定。

------------------------------------------------------------------------

## 2. Prompt 分类

-   System Prompt
-   DataFrame Profile Prompt
-   User Prompt
-   Code Generation Prompt
-   Result Explanation Prompt
-   Chart Recommendation Prompt
-   Error Fix Prompt

------------------------------------------------------------------------

## 3. System Prompt

System Prompt 定义模型角色。

示例：

> 你是一名资深 Python 数据分析工程师。 你只能生成 Pandas 分析代码。
> 所有计算必须由 Python 完成。 不输出解释，不输出
> Markdown，不输出代码块。 最终必须将分析结果赋值给变量 result。

------------------------------------------------------------------------

## 4. DataFrame Profile

发送给模型的信息包括：

-   行数
-   列数
-   字段名称
-   字段类型
-   缺失值统计
-   前 5\~10 行样例

不要发送完整 DataFrame。

------------------------------------------------------------------------

## 5. User Prompt

仅包含：

-   用户问题
-   必要上下文

不要重复发送 Schema。

------------------------------------------------------------------------

## 6. Code Generation Prompt

要求：

-   输出纯 Python。
-   默认使用变量 df。
-   最终结果保存到 result。
-   不允许 print()。
-   不允许 input()。
-   不允许网络访问。
-   不允许文件读写。

示例：

用户： 按部门统计平均工资。

模型：

result = df.groupby("部门")\["工资"\].mean().reset_index()

------------------------------------------------------------------------

## 7. Chart Prompt

如果用户需要图表：

模型返回：

-   图表类型
-   x 字段
-   y 字段
-   title

实际图表由 Python 生成。

------------------------------------------------------------------------

## 8. Result Explanation Prompt

发送：

-   用户问题
-   Python 执行结果摘要

要求：

-   不重新计算。
-   不编造数据。
-   用自然语言总结主要发现。

------------------------------------------------------------------------

## 9. Error Fix Prompt

输入：

-   原始代码
-   错误信息

要求：

-   修复代码
-   保持业务逻辑不变
-   返回完整可执行代码

------------------------------------------------------------------------

## 10. Few-shot 示例

建议维护：

-   排序
-   筛选
-   分组统计
-   Top N
-   日期分析
-   图表生成

作为 Few-shot 示例库。

------------------------------------------------------------------------

## 11. Token 控制

优先发送：

1.  Schema
2.  Sample
3.  用户问题

避免重复上下文。

------------------------------------------------------------------------

## 12. 输出规范

默认：

-   Python 代码：纯文本
-   解释：自然语言
-   图表：JSON/结构化描述

禁止混合多种输出格式。

------------------------------------------------------------------------

## 13. 安全约束

禁止模型生成：

-   os
-   subprocess
-   socket
-   requests
-   shutil
-   eval
-   exec
-   open

所有代码进入 Safe Executor 前必须经过 AST 校验。

------------------------------------------------------------------------

## 14. 可维护性

所有 Prompt 建议保存在：

prompts/

每个 Prompt 独立文件，支持版本管理、A/B 测试和统一维护。
